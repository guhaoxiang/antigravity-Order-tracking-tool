# 雲端部署指南 — Google Cloud Run

每次將新服務部署到 Cloud Run 時，依照此指南操作。

---

## 一、部署前需要準備的資料

| 項目 | 說明 | 範例 |
|------|------|------|
| **GCP Project ID** | Google Cloud 專案 ID | `zemo-h` |
| **服務名稱** | Cloud Run 上的服務名稱 | `slot-monitor` |
| **區域** | 部署的 GCP 區域 | `asia-east1`（台灣） |
| **排程頻率** | Cloud Scheduler 的 cron 表達式 | `*/30 * * * *`（每 30 分鐘） |
| **環境變數** | 所有 `.env` 中的 key-value | 見 `.env.example` |
| **費用上限通知信箱** | 收取預算警報的 email | `service@zemotw.com` |
| **每月預算金額 (USD)** | 超過此金額時發送通知 | `10` |

## 二、首次部署步驟

### 步驟 1：確認 gcloud CLI 已登入

```bash
gcloud auth login
gcloud config set project <PROJECT_ID>

# 啟用必要的 API
gcloud services enable run.googleapis.com
gcloud services enable cloudbuild.googleapis.com
gcloud services enable cloudscheduler.googleapis.com
gcloud services enable artifactregistry.googleapis.com
gcloud services enable billingbudgets.googleapis.com
```

### 步驟 2：建立 Artifact Registry（Docker 映像倉庫）

```bash
gcloud artifacts repositories create docker-repo \
  --repository-format=docker \
  --location=asia-east1 \
  --description="Docker images"
```

### 步驟 3：建構並推送 Docker 映像

```bash
# 設定 Docker 認證
gcloud auth configure-docker asia-east1-docker.pkg.dev

# 在專案根目錄建構
docker build -t asia-east1-docker.pkg.dev/<PROJECT_ID>/docker-repo/<SERVICE_NAME>:latest .

# 推送
docker push asia-east1-docker.pkg.dev/<PROJECT_ID>/docker-repo/<SERVICE_NAME>:latest
```

### 步驟 4：部署到 Cloud Run

```bash
gcloud run deploy <SERVICE_NAME> \
  --image=asia-east1-docker.pkg.dev/<PROJECT_ID>/docker-repo/<SERVICE_NAME>:latest \
  --region=asia-east1 \
  --platform=managed \
  --no-allow-unauthenticated \
  --memory=512Mi \
  --cpu=1 \
  --timeout=300 \
  --max-instances=1 \
  --min-instances=0 \
  --set-env-vars="SUPABASE_URL=<值>,SUPABASE_KEY=<值>,SLACK_BOT_TOKEN=<值>,SLACK_CHANNEL_ID=<值>,DAYS_AHEAD=30,SALES_START_HOUR=4,PROCESS_TIMEOUT_MS=300000,AUTH_EMAIL=<值>,AUTH_PASSWORD=<值>,SESSION_SECRET=<值>,ZEMO_API_BASE_URL=<值>"
```

> **重要參數解釋：**
> - `--max-instances=1`：防止多個實例同時執行（避免重複通知）
> - `--timeout=300`：5 分鐘逾時（與程式內部逾時一致）
> - `--no-allow-unauthenticated`：不開放公開存取（由 Cloud Scheduler 觸發）
> - `--min-instances=0`：閒置時不消耗費用

### 步驟 5：設定 Cloud Scheduler（定時排程）

```bash
# 建立 Service Account 給 Scheduler 用
gcloud iam service-accounts create scheduler-invoker \
  --display-name="Cloud Scheduler Invoker"

# 授權 Service Account 可以呼叫 Cloud Run
gcloud run services add-iam-policy-binding <SERVICE_NAME> \
  --region=asia-east1 \
  --member="serviceAccount:scheduler-invoker@<PROJECT_ID>.iam.gserviceaccount.com" \
  --role="roles/run.invoker"

# 取得 Cloud Run 的 URL
SERVICE_URL=$(gcloud run services describe <SERVICE_NAME> --region=asia-east1 --format='value(status.url)')

# 建立排程（每 30 分鐘執行一次）
gcloud scheduler jobs create http <SERVICE_NAME>-trigger \
  --location=asia-east1 \
  --schedule="*/30 * * * *" \
  --uri="${SERVICE_URL}" \
  --http-method=GET \
  --oidc-service-account-email=scheduler-invoker@<PROJECT_ID>.iam.gserviceaccount.com \
  --attempt-deadline=300s \
  --max-retry-attempts=1 \
  --min-backoff-duration=60s \
  --time-zone="Asia/Taipei"
```

> **重試風暴防護：**
> - `--max-retry-attempts=1`：最多只重試 1 次
> - `--min-backoff-duration=60s`：重試前等待 60 秒
> - `--attempt-deadline=300s`：單次嘗試最多 5 分鐘

## 三、費用限制與預算警報

### 設定預算警報（每 $10 USD 通知一次）

```bash
# 取得 Billing Account ID
BILLING_ACCOUNT=$(gcloud billing accounts list --format='value(name)' --limit=1)

# 建立預算（每月 $10 USD 閾值）
gcloud billing budgets create \
  --billing-account=$BILLING_ACCOUNT \
  --display-name="<SERVICE_NAME> 預算警報" \
  --budget-amount=10.00USD \
  --threshold-rule=percent=1.0,basis=current-spend \
  --threshold-rule=percent=2.0,basis=current-spend \
  --threshold-rule=percent=3.0,basis=current-spend \
  --threshold-rule=percent=5.0,basis=current-spend \
  --all-updates-rule-monitoring-notification-channels=<CHANNEL_ID> \
  --filter-projects="projects/<PROJECT_ID>"
```

**或者用 Google Cloud Console UI 操作（比較簡單）：**

1. 前往 https://console.cloud.google.com/billing
2. 點選「預算與快訊」→「建立預算」
3. 名稱：`<服務名稱> 月費監控`
4. 金額：$10 USD
5. 閾值百分比：100%, 200%, 300%, 500%（= $10, $20, $30, $50 時通知）
6. 通知收件人：
   - `service@zemotw.com`
   - `haoxiangkuo@zemotw.com`
   - `qwa144699@gmail.com`
7. 勾選「電子郵件通知」

## 四、重試風暴防護清單

部署前確認以下防護措施已啟用：

| 防護措施 | 位置 | 設定值 |
|----------|------|--------|
| 全域逾時 | `slot-monitor/index.js` | `PROCESS_TIMEOUT_MS=300000`（5 分鐘） |
| 單日處理逾時 | `slot-monitor/index.js` | `PER_DATE_TIMEOUT=60000`（60 秒） |
| Slack 連續失敗上限 | `slot-monitor/core/slack-notifier.js` | 連續 3 次失敗後停止 |
| Slack 訊息數上限 | `slot-monitor/core/slack-notifier.js` | `MAX_MESSAGES=20` |
| Slack 重試次數 | `slot-monitor/core/slack-notifier.js` | 最多 1 次重試 |
| Cloud Run 最大實例 | Cloud Run 設定 | `--max-instances=1` |
| Scheduler 重試上限 | Cloud Scheduler 設定 | `--max-retry-attempts=1` |
| Scheduler 退避時間 | Cloud Scheduler 設定 | `--min-backoff-duration=60s` |

## 五、更新部署

程式碼修改後，重新建構並推送：

```bash
# 建構新映像
docker build -t asia-east1-docker.pkg.dev/<PROJECT_ID>/docker-repo/<SERVICE_NAME>:latest .

# 推送
docker push asia-east1-docker.pkg.dev/<PROJECT_ID>/docker-repo/<SERVICE_NAME>:latest

# 更新 Cloud Run（會自動拉取新映像）
gcloud run deploy <SERVICE_NAME> \
  --image=asia-east1-docker.pkg.dev/<PROJECT_ID>/docker-repo/<SERVICE_NAME>:latest \
  --region=asia-east1
```

## 六、監控與除錯

```bash
# 查看 Cloud Run 日誌
gcloud run services logs read <SERVICE_NAME> --region=asia-east1 --limit=50

# 查看 Scheduler 執行紀錄
gcloud scheduler jobs describe <SERVICE_NAME>-trigger --location=asia-east1

# 手動觸發一次
gcloud scheduler jobs run <SERVICE_NAME>-trigger --location=asia-east1

# 暫停排程
gcloud scheduler jobs pause <SERVICE_NAME>-trigger --location=asia-east1

# 恢復排程
gcloud scheduler jobs resume <SERVICE_NAME>-trigger --location=asia-east1
```

## 七、費用估算

| 項目 | 每月估算 |
|------|---------|
| Cloud Run（每 30 分鐘跑一次，每次 ~15 秒，512MB） | ~$0.50 |
| Cloud Scheduler | 免費（前 3 個排程免費） |
| Artifact Registry（Docker 映像儲存） | ~$0.10 |
| **預估月費** | **< $1 USD** |

> 以每 30 分鐘執行一次、每次約 15 秒計算，Cloud Run 免費額度（每月 200 萬次請求 + 360,000 GB-秒）遠超此用量，實際費用可能為 $0。
