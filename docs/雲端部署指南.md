# 雲端部署指南 — Google Cloud Run

本專案部署兩個服務到 Cloud Run：
1. **slot-monitor**：暫停銷售自動通知（每 30 分鐘由 Cloud Scheduler 觸發）
2. **scheduler-web-demo**：排程設定網頁介面（常駐 HTTP 服務）

兩者共用同一份程式碼，由 `server.js` 的 `SERVICE_MODE` 環境變數決定啟動哪個。

---

## 一、部署前需要準備的資料

| 項目 | 說明 | 範例 |
|------|------|------|
| **GCP Project ID** | Google Cloud 專案 ID | `zemo-h` |
| **區域** | 部署的 GCP 區域 | `asia-east1`（台灣） |
| **環境變數** | 見 `.env.example` | Supabase、Slack、Zemo API 等 |
| **費用上限通知信箱** | 收取預算警報的 email | `service@zemotw.com` |
| **每月預算金額 (USD)** | 超過此金額時發送通知 | `10` |

## 二、首次部署步驟

### 步驟 1：確認 gcloud CLI 已登入

```bash
gcloud auth login
gcloud config set project zemo-h

# 啟用必要的 API
gcloud services enable run.googleapis.com
gcloud services enable cloudbuild.googleapis.com
gcloud services enable cloudscheduler.googleapis.com
gcloud services enable artifactregistry.googleapis.com
gcloud services enable billingbudgets.googleapis.com
```

### 步驟 2：建立 Artifact Registry（Docker 映像倉庫）

```bash
gcloud artifacts repositories create docker-repo \
  --repository-format=docker \
  --location=asia-east1 \
  --description="Docker images"
```

### 步驟 3：建構並推送 Docker 映像

```bash
# 設定 Docker 認證
gcloud auth configure-docker asia-east1-docker.pkg.dev

# 在專案根目錄建構
docker build -t asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest .

# 推送
docker push asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest
```

### 步驟 4：部署 slot-monitor（Slack 定時通知）

```bash
gcloud run deploy slot-monitor \
  --image=asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest \
  --region=asia-east1 \
  --platform=managed \
  --no-allow-unauthenticated \
  --memory=512Mi \
  --cpu=1 \
  --timeout=300 \
  --max-instances=1 \
  --min-instances=0 \
  --set-env-vars="SUPABASE_URL=<值>,SUPABASE_KEY=<值>,SLACK_BOT_TOKEN=<值>,SLACK_CHANNEL_ID=<值>,DAYS_AHEAD=30,SALES_START_HOUR=4,PROCESS_TIMEOUT_MS=300000,ZEMO_API_BASE_URL=<值>,ZEMO_LINE_ID=<值>"
```

### 步驟 5：部署 scheduler-web-demo（網頁介面）

使用同一個映像，只改 `SERVICE_MODE`：

```bash
gcloud run deploy scheduler-web \
  --image=asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest \
  --region=asia-east1 \
  --platform=managed \
  --allow-unauthenticated \
  --memory=512Mi \
  --cpu=1 \
  --timeout=300 \
  --max-instances=2 \
  --min-instances=0 \
  --set-env-vars="SERVICE_MODE=web,SUPABASE_URL=<值>,SUPABASE_KEY=<值>,AUTH_EMAIL=<值>,AUTH_PASSWORD=<值>,SESSION_SECRET=<值>,ZEMO_API_BASE_URL=<值>,ZEMO_LINE_ID=<值>,SLACK_BOT_TOKEN=<值>,SLACK_CHANNEL_ID=<值>"
```

> **兩個服務的差異：**
>
> | | slot-monitor | scheduler-web |
> |---|---|---|
> | SERVICE_MODE | 不設（預設 monitor） | `web` |
> | 觸發方式 | Cloud Scheduler 每 30 分鐘 | 使用者開瀏覽器 |
> | 公開存取 | 否（`--no-allow-unauthenticated`） | 是（有登入頁面保護） |
> | max-instances | 1（防重複） | 2 |

### 步驟 6：設定 Cloud Scheduler（定時排程）

```bash
# 建立 Service Account 給 Scheduler 用
gcloud iam service-accounts create scheduler-invoker \
  --display-name="Cloud Scheduler Invoker"

# 授權 Service Account 可以呼叫 Cloud Run
gcloud run services add-iam-policy-binding slot-monitor \
  --region=asia-east1 \
  --member="serviceAccount:scheduler-invoker@zemo-h.iam.gserviceaccount.com" \
  --role="roles/run.invoker"

# 取得 Cloud Run 的 URL
SERVICE_URL=$(gcloud run services describe slot-monitor --region=asia-east1 --format='value(status.url)')

# 建立排程（每 30 分鐘執行一次）
gcloud scheduler jobs create http slot-monitor-trigger \
  --location=asia-east1 \
  --schedule="*/30 * * * *" \
  --uri="${SERVICE_URL}" \
  --http-method=GET \
  --oidc-service-account-email=scheduler-invoker@zemo-h.iam.gserviceaccount.com \
  --attempt-deadline=300s \
  --max-retry-attempts=1 \
  --min-backoff-duration=60s \
  --time-zone="Asia/Taipei"
```

> **重試風暴防護：**
> - `--max-retry-attempts=1`：最多只重試 1 次
> - `--min-backoff-duration=60s`：重試前等待 60 秒
> - `--attempt-deadline=300s`：單次嘗試最多 5 分鐘

## 三、費用限制與預算警報

**用 Google Cloud Console UI 操作（最簡單）：**

1. 前往 https://console.cloud.google.com/billing
2. 點選「預算與快訊」→「建立預算」
3. 名稱：`Antigravity 單量追蹤 月費監控`
4. 金額：$10 USD
5. 閾值百分比：100%, 200%, 300%, 500%（= $10, $20, $30, $50 時通知）
6. 通知收件人：填入你的 email
7. 勾選「電子郵件通知」

## 四、重試風暴防護清單

部署前確認以下防護措施已啟用：

| 防護措施 | 位置 | 設定值 |
|----------|------|--------|
| 全域逾時 | `slot-monitor/index.js` | `PROCESS_TIMEOUT_MS=300000`（5 分鐘） |
| 單日處理逾時 | `slot-monitor/index.js` | `PER_DATE_TIMEOUT=60000`（60 秒） |
| Slack 訊息數上限 | `slot-monitor/core/slack-notifier.js` | `MAX_MESSAGES=20` |
| Slack 重試次數 | `slot-monitor/core/slack-notifier.js` | 最多 1 次重試 |
| Cloud Run 最大實例 | Cloud Run 設定 | `--max-instances=1` |
| Scheduler 重試上限 | Cloud Scheduler 設定 | `--max-retry-attempts=1` |
| Scheduler 退避時間 | Cloud Scheduler 設定 | `--min-backoff-duration=60s` |

## 五、更新部署

程式碼修改後，重新建構並推送：

```bash
# 建構新映像
docker build -t asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest .

# 推送
docker push asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest

# 更新 slot-monitor
gcloud run deploy slot-monitor \
  --image=asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest \
  --region=asia-east1

# 更新 scheduler-web（同一個映像）
gcloud run deploy scheduler-web \
  --image=asia-east1-docker.pkg.dev/zemo-h/docker-repo/slot-monitor:latest \
  --region=asia-east1
```

## 六、監控與除錯

```bash
# 查看 slot-monitor 日誌
gcloud run services logs read slot-monitor --region=asia-east1 --limit=50

# 查看 scheduler-web 日誌
gcloud run services logs read scheduler-web --region=asia-east1 --limit=50

# 查看 Scheduler 執行紀錄
gcloud scheduler jobs describe slot-monitor-trigger --location=asia-east1

# 手動觸發一次 slot-monitor
gcloud scheduler jobs run slot-monitor-trigger --location=asia-east1

# 暫停排程
gcloud scheduler jobs pause slot-monitor-trigger --location=asia-east1

# 恢復排程
gcloud scheduler jobs resume slot-monitor-trigger --location=asia-east1
```

## 七、費用估算

| 項目 | 每月估算 |
|------|---------|
| Cloud Run — slot-monitor（每 30 分鐘跑一次，每次 ~15 秒，512MB） | ~$0.50 |
| Cloud Run — scheduler-web（偶爾使用，min-instances=0） | ~$0.10 |
| Cloud Scheduler | 免費（前 3 個排程免費） |
| Artifact Registry（Docker 映像儲存） | ~$0.10 |
| **預估月費** | **< $1 USD** |

> 以 Cloud Run 免費額度（每月 200 萬次請求 + 360,000 GB-秒）來看，此用量遠低於免費額度，實際費用可能為 $0。

## 八、架構圖

```
Cloud Scheduler (每30分鐘)
    │
    ▼ HTTP GET
┌─────────────────────┐
│ Cloud Run            │
│ slot-monitor         │  ← SERVICE_MODE 不設
│ server.js            │
│   └→ slot-monitor/   │  → 計算暫停銷售段
│       └→ Slack 通知  │  → 差異比對 → 發送通知
│       └→ Supabase    │  → 儲存狀態
└─────────────────────┘

使用者瀏覽器
    │
    ▼ HTTP
┌─────────────────────┐
│ Cloud Run            │
│ scheduler-web        │  ← SERVICE_MODE=web
│ server.js            │
│   └→ scheduler-web-  │  → 設定頁面 + 排程模擬
│      demo/app.js     │  → Supabase 存設定
└─────────────────────┘

兩個服務共用：
  ├── scheduler-web-demo/zemo-lib/  ← 排程演算法核心
  ├── Supabase                      ← 設定 + 通知狀態
  └── Zemo API (zemotw.com)         ← 預約 + 駕駛資料
```
